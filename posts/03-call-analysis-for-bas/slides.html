The problem

Sound familiar?

  • You review the same kind of meeting notes every week — manually
  • Preparing a status update takes longer than the actual work
  • You think AI could help but "that's for developers"

I said that last one for two years. Then I tried Claude Code.

02 / 08
The tool

What is Claude Code?

A terminal-based AI agent that reads your files, runs tasks, and saves results — in plain English.

npm install -g @anthropic/claude-code
cd your-project
claude

No coding required. If you can write a clear email, you can use this.

03 / 08
The framework

Every prompt has
the same structure.

Take

What data?

Files, folders, text, a URL

Process

What to do?

Find, classify, summarise, compare

Store

Where to save?

A file, a report, a folder

This is the same thinking BAs already do. The agent just executes it.

04 / 08
Real example

My call analysis prompt.

In the calls/ folder are 11 meeting transcripts.
Read all of them. Find repeating problems.
For each: what it is, who raised it, impact.
Save a structured report to report.md.

I turned this into a reusable Skill — one command, every time:

/call-analysis
05 / 08
The result

The actual difference.

Before

Read 11 transcripts manually, take notes, find patterns, format a report.

~3 hrs
After

Type /call-analysis in the terminal. Review the output.

~8 min

The thinking is still mine. The reading and formatting is the agent's.

06 / 08
Get started today

3 steps to try this.

  1. Install Claude Code: npm install -g @anthropic/claude-code
  2. Copy any skill folder from GitHub into .claude/skills/ in your project
  3. Restart Claude Code, type /call-analysis and press Enter
