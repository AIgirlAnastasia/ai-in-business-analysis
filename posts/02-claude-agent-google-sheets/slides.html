The problem

Two wrong ways to use AI for data.

❌ Ask Claude to count directly

"What's the conversion rate in my 3,000-row sheet?"

"Sure! It's around 34%..." ← wrong number, sounds confident

✓ Ask Claude to write the count

"Write a script that calculates the conversion rate from this CSV."

Script runs → 98.2% ← exact, every time

LLMs are good at language. Code is good at math. Use them for what they're built for.

02 / 08
The concept

The Module Pattern:
Code counts. LLM interprets.

You don't need to learn Python.
You need to learn to ask the right question.

Claude writes the logic. The script runs the math. You stay in the middle — asking questions, making decisions.

This is exactly what a senior analyst does: they don't manually count rows. They build systems that count, then interpret the output.

03 / 08
The 4-step loop

Same pattern. Every analysis.

Step 1 — Get

Connect data

Google Sheets via MCP. Claude reads the sheet directly.

Step 2 — Write

Ask Claude

Describe the question. Claude writes the formula or script.

Step 3 — Run

Execute

Code runs, not LLM. Numbers are exact. No hallucination.

Step 4 — Explain

Interpret

Paste output back. Claude explains the pattern and flags anomalies.

The split: Claude handles language + logic. Code handles arithmetic. You handle judgment.

05 / 08
Live example — the script Claude wrote

I wrote zero lines of this.

from collections import defaultdict, Counter
from datetime import datetime
import csv

with open('data.csv') as f:
    rows = list(csv.DictReader(f))

monthly = defaultdict(Counter)
for r in rows:
    dt = datetime.strptime(r['Time stamp'], '%d/%m/%Y %H:%M')
    monthly[dt.strftime('%Y-%m')][r['Interaction type']] += 1

for month in sorted(monthly):
    c = monthly[month]
    print(f"{month}: purchases={c['purchase']}")

I only wrote the question. Claude wrote this. Then I ran it and got exact monthly numbers.

06 / 08
Step 4 — explain

Script ran. I asked Claude:
"What does this mean?"

  • 71.5% of the product catalog was never purchased — ever
  • Jan 2023 and Oct–Dec 2023 are clear peak months — seasonal pattern
  • February to September is flat — possible marketing gap

This took 5 minutes. The same analysis in Excel would have taken an hour — and I'd still question the numbers.

The code counted. Claude interpreted. I made the decision.

How to start

One command to connect
Claude to Google Sheets.

  1. Install the MCP server:
    claude mcp add google-sheets-mcp -- npx -y mcp-google-sheets
  2. Authenticate — Claude opens the Google OAuth flow. Takes 2 minutes.
  3. Point Claude at your sheet and describe the question. It writes the formula or script.
